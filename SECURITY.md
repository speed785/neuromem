# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in neuromem, please report it privately.

- Preferred: create a private GitHub Security Advisory at `https://github.com/speed785/neuromem/security/advisories/new`
- Alternative: email the maintainer with details if private advisory is unavailable

Please include:

- A clear description of the issue and impact
- Reproduction steps or proof of concept
- Affected versions and environments
- Any suggested remediation

## Response process

- We will acknowledge receipt within 72 hours.
- We will investigate and validate the report.
- We will coordinate remediation and release timing.
- We will publish a fix and security notes once users can safely update.

Please do not open public issues for unpatched vulnerabilities.
